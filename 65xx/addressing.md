# アドレッシングモード

命令のオペランドから操作対象のアドレスを取得する方法です。

```
  ゼロページアドレッシング
    ダイレクトアドレッシングとも呼ばれます。
    8bitのパラメータ(1バイト)を使って、メモリの最初の256バイト 0000h...00FFh(=ゼロページ) 内のアドレスを指定します。

	メリット:
	  オペランドが1バイトで済むため、ROM容量の小さいこの時代には重宝されていたようです。
	デメリット:
	  ゼロページの範囲しか指定できない
```

```
  絶対アドレッシング
    16bitのパラメータ（2バイト）を使って、64KBのメモリ空間 0000h...FFFFh のアドレスのどこかを指定します。

	メリット:
	  どこでも指定可能
	デメリット:
	  パラメータとして余計に1バイト追加されるため、ゼロページより少し遅くなります。
```

```
  間接アドレッシング
    ゼロページの16bitパラメータを指し示す8bitパラメータを使用します。

	メリット:
	  ポインタの扱いが楽
	  スーファミとは関係ないが、RISCは固定長という性質上、少ないbit数でアドレスを指定したいことが多いので役に立つ
	デメリット:
	  メモリアクセスが余計に1回加わるため低速
```
