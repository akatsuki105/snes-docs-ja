# アドレッシングモード

命令のオペランドから操作対象のアドレスを取得する方法です。

```
  即値アドレッシング
    正確にはアドレッシングではないです。
    オペコードの直後のオペランドをそのまま使用します。

	メリット:
	  他のアドレッシングモードは、アドレス取得から対象読み出しでメモリ読み込みが必要だが、即値アドレッシングでは必要ないため高速
	デメリット:
	  命令サイズが大きくなる
```

```
  絶対アドレッシング
    16bitのパラメータを使って、64KBのメモリ空間 0000h...FFFFh のアドレスのどこかを指定します。(バンクはDB)
	(24bitのパラメータを使ってバンクを含めて任意のアドレスを指定することもできます)

	メリット:
	  どこでも指定可能
	デメリット:
	  パラメータとして余計に1バイト追加されるため、ゼロページより少し遅い
```

```
  ゼロページアドレッシング
    ダイレクトアドレッシングとも呼ばれます。
    8bitのパラメータ(1バイト)を使って、メモリの最初の256バイト 0000h...00FFh(=ゼロページ) 内のアドレスを指定します。
	絶対アドレッシングの亜種で、上位8bitが 0x00 (ゼロページ)で固定された代わりにパラメータが8bitになりました。

	メリット:
	  オペランドが1バイトで済む
	   フェッチが1バイト少ないので絶対アドレッシングより高速
	   カセット容量の小さいこの時代には1バイトでも小さいことが重宝されていた
	デメリット:
	  ゼロページの範囲しか指定できない
```

```
  間接アドレッシング
	オペランドで指定したアドレスに真のアドレスが入っています。メモリを中継するので正確には メモリ間接(アドレッシング) とも呼びます。
    スーファミの場合は、ゼロページアドレッシングで取得したアドレスから16bit値を読み出し、それをアドレスとします。

	メリット:
	  ポインタの扱いが楽
	  スーファミとは関係ないが、RISCは固定長という性質上、少ないbit数でアドレスを指定したいことが多いので役に立つ
	デメリット:
	  メモリアクセスが余計に1回加わるため低速
```

```
  指標アドレッシング
    インデックスアドレッシングとも呼ばれます。
    他のアドレッシングモードで算出したアドレスに、インデックスレジスタ(X or Y)を足したものをアドレスとします。

	メリット:
	  配列などのメモリ上で連続したデータを処理するときに、インデックスレジスタを増やすことでループ上の同じ命令で配列を処理することができる
	デメリット:
	  なし(元になったアドレッシングモードに依存)
```
